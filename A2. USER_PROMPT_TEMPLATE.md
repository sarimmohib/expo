## What I care about most

1. **The buttons must work.** If the APK installs but buttons don't respond, that's the Turbopack getAssetPrefix bug — run `fix-asset-prefix-check.py` and rebuild.
2. **The error console must be usable.** I'm on a phone, I can't use chrome://inspect. The red banner + tappable modal + "Copy all" button is how I'll debug.
3. **Offline must actually work.** After first launch, the app should work with no network. GitHub sync is the only feature that needs network.
4. **The APK must be installable via sideloading.** No app store, no Expo Go dependency. Just: tap the APK, install, open.

## What I don't care about

- iOS support (for now)
- App store optimization
- Analytics / telemetry (don't add any)
- Multi-language UI (English only for now)
- Animated splash screen (the default Expo splash is fine)

## How to use this template

1. **Copy everything from "I need you to wrap..." to "Begin."**
2. **Fill in the `[BRACKETS]`:**
   - `[SOURCE_ARCHIVE.tar]` — the actual filename in `/upload/`
   - `[VERSION]` — Next.js version (e.g. 16.1.1)
   - `[NOTABLE FEATURES]` — list the main libs (helps the agent anticipate gotchas)
   - `[N]` API routes — count and brief description
   - `[APP_NAME]` — what to name the output APK
   - `[MIN_VERSION]` — minimum Android version (usually 10+)
   - Time budget `[N]` minutes — usually 30-60 for a baseline, 90-180 with features
3. **Adjust the Features section:** Delete waves you don't want. Add specific §-references for features you want that aren't listed. Or delete the entire Features section for a baseline build.
4. **Adjust "What I care about most" and "What I don't care about"** to match your actual priorities.
5. **Paste as the first user message** in a session where `SYSTEM_PROMPT.md` is already loaded as the system prompt.
