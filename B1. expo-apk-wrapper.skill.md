## Overview

This skill wraps a server-rendered Next.js PWA into an installable Android APK by:

1. Converting the Next.js app to a static HTML export (`output: "export"`)
2. Bundling the entire export as a TypeScript data module (`web-bundle.ts`)
3. Creating a minimal Expo + React Native WebView shell that unpacks the bundle to `FileSystem.documentDirectory` at first launch
4. Compiling a release APK via Gradle
5. Patching the Turbopack runtime so React hydrates correctly under a `file://` origin
6. (Optional) Adding any of 69 production features documented in the runbook

The result is a single `.apk` file the user can sideload — no app store required, no Expo Go dependency, fully offline-capable.

## The 5 bugs you WILL hit

These are not theoretical — every one was a real failure.

### Bug 1 — Buttons don't work, only scrolling

**Symptom:** APK installs, page renders, can scroll, but NO buttons respond.

**Cause:** The Turbopack runtime's `getAssetPrefix()` throws an `InvariantError` because `URL.pathname` never contains `./_next/` (only `_next/`). React never hydrates.

**Fix:** Run `fix-asset-prefix-check.py` (script below). It patches `indexOf("./_next/")` → `indexOf("_next/")` in the chunk that contains `getAssetPrefix`.

### Bug 2 — `cp: cannot create directory '.next/standalone/.next/'`

**Cause:** The `build` script in `package.json` has post-build copy steps assuming `output: "standalone"`.

**Fix:** Edit `package.json`:
```json
"build": "next build"
```

### Bug 3 — `ENOENT: no such file or directory, open './assets/adaptive-icon.png'`

**Cause:** `app.json` references icon files that don't exist.

**Fix:** Create the PNGs (Step 9) or remove the icon fields from `app.json`.

### Bug 4 — CMake Error: `add_subdirectory given source which is not an existing directory`

**Cause:** React Native New Architecture requires codegen for react-native-webview, but the codegen output directory doesn't exist.

**Fix:** Disable New Architecture:
```bash
sed -i 's/newArchEnabled=true/newArchEnabled=false/g' android/gradle.properties
sed -i 's/"newArchEnabled": true/"newArchEnabled": false/g' app.json
rm -rf android
./node_modules/.bin/expo prebuild --platform android --clean --no-install
echo "sdk.dir=$ANDROID_HOME" > android/local.properties
```

### Bug 5 — `D8: java.nio.file.FileSystemException: No space left on device`

**Cause:** Sandbox has ~10 GB disk. Gradle + NDK + node_modules exceeds this.

**Fix:** Aggressive cleanup (run whenever disk < 500 MB free):
```bash
rm -rf /home/z/.gradle/caches/8.13/transforms
rm -rf /home/z/.gradle/caches/8.13/generated-gradle-jars
rm -rf /home/z/.gradle/daemon
rm -rf /home/z/my-project/work/node_modules
rm -rf /home/z/my-project/work/.next
rm -rf /home/z/my-project/work/out
rm -rf /home/z/my-project/expo-app/node_modules/*/android/build
rm -rf /home/z/my-project/expo-app/android/app/build
```

**Do NOT delete:**
- `/home/z/.gradle/caches/modules-2/` (dependency cache, slow to rebuild)
- `/home/z/.gradle/wrapper/dists/gradle-8.13-bin/` (gradle distribution)
- `/home/z/my-project/expo-app/src/web-bundle.ts` (your app)

## On-device debugging (critical)

Users on phones can't use `chrome://inspect`. Build the debugging UI into the app:

1. Inject a bootstrap script via `injectedJavaScriptBeforeContentLoaded` that captures `window.onerror`, `unhandledrejection`, and `console.error`, and forwards them to native via `window.ReactNativeWebView.postMessage`.
2. On the native side, `onMessage` handler appends to a `jsErrors` state array (cap at 50).
3. A red banner at the bottom shows the latest error in FULL (no truncation).
4. Tapping the banner opens a full-screen `<Modal>` with a `<ScrollView>` showing ALL errors in monospace.
5. "Copy all" button uses `expo-clipboard` to copy the entire log to the system clipboard.

This is non-negotiable for shipping WebView apps to non-technical users. See the runbook §10 and §26 for the full implementation.

## Anti-patterns (don't do these)

1. Don't ship all 69 features at once. Pick a wave, ship, iterate.
2. Don't add features without Settings toggles. Every behavioral change should be opt-out.
3. Don't duplicate PWA features in native. If the PWA already does it (e.g. dark mode toggle), bridge — don't rebuild.
4. Don't add native features that the PWA can do. Most "polish" features are better in the PWA's React code.
5. Don't add `expo-background-fetch` without testing on real Android devices. Doze mode will mock your scheduled tasks.
6. Don't add biometric lock without a fallback. Devices without biometrics need a passcode option.
7. Don't trust `nohup` in sandboxes. Use `setsid` or foreground with long tool timeout.
8. Don't use the global `expo-cli` npm package. It's deprecated. Use `./node_modules/.bin/expo`.
9. Don't promise an Expo Go URL in sandboxes. ngrok connects but the dev server dies. Build the APK.
10. Don't use `expo-asset` for folders of `.js` files. Metro parses them as JS modules. Use the data-module approach.

## Reference scripts (embedded)

### scripts/build-web-bundle.py

```python
#!/usr/bin/env python3
"""
Build a TypeScript data module from a Next.js static export.
Reads:  /home/z/my-project/work/out/
Writes: /home/z/my-project/expo-app/src/web-bundle.ts
"""
import base64
import json
import re
from pathlib import Path

SOURCE_DIR = Path("/home/z/my-project/work/out")
OUTPUT_FILE = Path("/home/z/my-project/expo-app/src/web-bundle.ts")

SKIP_PATTERNS = [
    re.compile(r"^__next\."),
    re.compile(r"\.txt$"),
    re.compile(r"^404($|/)"),
    re.compile(r"^_not-found($|/)"),
    re.compile(r"^robots\.txt$"),
]

def should_skip(rel_path):
    return any(p.search(rel_path) for p in SKIP_PATTERNS)

def rewrite_paths(content):
    content = re.sub(r'((?:src|href)=["\'])/_next/', r"\1./_next/", content)
    content = re.sub(r'((?:src|href)=["\'])/(?!/)', r"\1./", content)
    content = re.sub(r'url\(["\']?/_next/', "url(./_next/", content)
    content = re.sub(r'"/_next/', '"./_next/', content)
    return content

def main():
    if not SOURCE_DIR.exists():
        raise SystemExit(f"Source directory not found: {SOURCE_DIR}")

    files = {}
    total_bytes = 0
    skipped = 0

    for root, dirs, filenames in SOURCE_DIR.walk():
        dirs[:] = [d for d in dirs if not d.startswith(".") and not d.startswith("__")]
        for filename in filenames:
            full_path = root / filename
            rel_path = str(full_path.relative_to(SOURCE_DIR)).replace("\\", "/")
            if should_skip(rel_path):
                skipped += 1
                continue
            data = full_path.read_bytes()
            total_bytes += len(data)
            try:
                text = data.decode("utf-8")
                if rel_path.endswith(".html") or rel_path.endswith(".htm"):
                    text = rewrite_paths(text)
                files[rel_path] = {"type": "text", "content": text}
            except UnicodeDecodeError:
                files[rel_path] = {
                    "type": "base64",
                    "content": base64.b64encode(data).decode("ascii"),
                }

    OUTPUT_FILE.parent.mkdir(parents=True, exist_ok=True)
    with OUTPUT_FILE.open("w", encoding="utf-8") as f:
        f.write("// AUTO-GENERATED — DO NOT EDIT.\n")
        f.write("// Produced by scripts/build-web-bundle.py\n\n")
        f.write("export interface WebBundleFile {\n  type: 'text' | 'base64';\n  content: string;\n}\n\n")
        f.write("export const WEB_BUNDLE: Record<string, WebBundleFile> = ")
        f.write(json.dumps(files, ensure_ascii=False, separators=(",", ":")))
        f.write(";\n")

    print(f"Wrote {OUTPUT_FILE}")
    print(f"  Files: {len(files)} (skipped {skipped})")
    print(f"  Source bytes: {total_bytes:,}")
    print(f"  Output size: {OUTPUT_FILE.stat().st_size:,} bytes")

if __name__ == "__main__":
    main()
```

### scripts/fix-asset-prefix-check.py

```python
#!/usr/bin/env python3
"""
Patch the getAssetPrefix InvariantError that breaks React hydration
when the PWA runs from a file:// origin inside the WebView.

Patches indexOf("./_next/") → indexOf("_next/") in the chunk that
contains 'Expected document.currentScript src to contain'.
"""
import json
import re
from pathlib import Path

BUNDLE = Path("/home/z/my-project/expo-app/src/web-bundle.ts")

def find_bundle_object(text):
    m = re.search(r"WEB_BUNDLE[^=]*=\s*", text)
    start = m.end()
    i = start
    depth = 0
    in_str = False
    str_ch = None
    escape = False
    while i < len(text):
        ch = text[i]
        if in_str:
            if escape: escape = False
            elif ch == "\\": escape = True
            elif ch == str_ch: in_str = False
        else:
            if ch in ('"', "'", "`"): in_str, str_ch = True, ch
            elif ch == "{": depth += 1
            elif ch == "}":
                depth -= 1
                if depth == 0: return start, i + 1
        i += 1
    raise RuntimeError("Bundle object not found")

def main():
    text = BUNDLE.read_text()
    start, end = find_bundle_object(text)
    data = json.loads(text[start:end])

    patched = 0
    for k, v in data.items():
        if k.endswith(".js") and "Expected document.currentScript src to contain" in v.get("content", ""):
            original = v["content"]
            v["content"] = original.replace('t.indexOf("./_next/")', 't.indexOf("_next/")')
            if v["content"] != original:
                patched += 1
                print(f"  patched: {k}")

    if patched == 0:
        print("No patches applied (already patched or pattern not found).")
        return

    new_bundle_str = json.dumps(data, ensure_ascii=False, separators=(",", ":"))
    new_text = text[:start] + new_bundle_str + text[end:]
    BUNDLE.write_text(new_text)
    print(f"\nPatched {patched} chunk(s). Wrote: {BUNDLE}")

if __name__ == "__main__":
    main()
```

## Stop conditions (when to ask the user)

Stop and ask the user before proceeding if:

1. The Next.js source has database / filesystem access in API routes that can't be moved client-side.
2. The source uses Next.js features that can't be statically exported (e.g. ISR, SSR-only pages, image optimization).
3. Disk space is below 2 GB and you've already cleaned everything safe to clean.
4. The user's source tree doesn't have a `package.json` or `next.config.{ts,js}`.
5. The build fails for a reason not covered in the bugs section above, and you've spent >30 minutes trying to fix it.
6. The user requests a feature not in the catalog (§18-§32).

In all other cases, proceed autonomously and report progress in `worklog.md`.
