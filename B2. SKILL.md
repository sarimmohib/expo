# SKILL.md — Expo APK Wrapper

> **Description:** Wrap a Next.js / React PWA into an installable Android APK using Expo + React Native WebView. Trigger when the user wants to ship an installable mobile app from a web/PWA source, mentions Expo / React Native / WebView shell / native wrapper / offline mobile app / Android packaging, or wants native mobile features (haptics, notifications, biometrics, gestures, file system) added to a web app.
>
> **Auto-trigger keywords:** expo, apk, aab, "wrap pwa", "wrap nextjs", "react native webview", "mobile app", "native shell", "installable app", "offline app", "android app", "ios app", "pwa to apk", "pwa to app", "web to mobile", "web to native", "expo prebuild", "gradle assembleRelease", turbopack, "file:// webview"
>
> **Do NOT trigger for:** pure Next.js web development (no mobile target), building a native app from scratch (no existing web app), Capacitor/Cordova/Tauri wrapping.

---

## What this skill does

Takes an existing Next.js (or any static-exportable) PWA and produces an installable Android APK by:

1. Converting the Next.js app to static HTML (`output: "export"`)
2. Bundling the entire export as a TypeScript data module (~3.5 MB → `web-bundle.ts`)
3. Wrapping it in a minimal Expo + React Native WebView shell
4. Compiling a release APK via Gradle
5. Patching the Turbopack runtime so React hydrates under `file://` origin
6. Optionally adding any of 69 production features (haptics, gestures, notifications, biometrics, etc.)

Output: a single `.apk` the user can sideload — no app store required, no Expo Go dependency, fully offline.

## The 5 bugs you WILL hit

### Bug 1: Buttons don't work, only scrolling
**Cause:** Turbopack's `getAssetPrefix()` throws `InvariantError` because `URL.pathname` never contains `./_next/`. React never hydrates.
**Fix:** Run `fix-asset-prefix-check.py` (patches `indexOf("./_next/")` → `indexOf("_next/")`).

### Bug 2: `cp: cannot create directory '.next/standalone/.next/'`
**Cause:** `package.json` build script has stale `cp -r` from `output: "standalone"`.
**Fix:** `"build": "next build"` (remove the cp parts).

### Bug 3: `ENOENT: ... assets/adaptive-icon.png`
**Cause:** `app.json` references icons that don't exist.
**Fix:** Create the PNGs or remove the icon fields.

### Bug 4: CMake `add_subdirectory given source which is not an existing directory`
**Cause:** New Architecture codegen output for `react-native-webview` is missing.
**Fix:** Disable New Arch — `newArchEnabled=false` in both `app.json` and `android/gradle.properties`. Re-prebuild.

### Bug 5: `D8: No space left on device`
**Cause:** Sandbox has ~10 GB. Gradle + NDK + node_modules exceeds this.
**Fix:** Aggressive cleanup:
```bash
rm -rf /home/z/.gradle/caches/8.13/transforms
rm -rf /home/z/.gradle/caches/8.13/generated-gradle-jars
rm -rf /home/z/.gradle/daemon
rm -rf /home/z/my-project/work/node_modules
rm -rf /home/z/my-project/work/.next /home/z/my-project/work/out
rm -rf /home/z/my-project/expo-app/node_modules/*/android/build
rm -rf /home/z/my-project/expo-app/android/app/build
```
**Do NOT delete:** `~/.gradle/caches/modules-2/`, `~/.gradle/wrapper/dists/`, `expo-app/src/web-bundle.ts`.

## On-device debugging (critical)

Users on phones can't use `chrome://inspect`. Build the debugging UI into the app:

1. Inject `injectedJavaScriptBeforeContentLoaded` that captures `window.onerror`, `unhandledrejection`, `console.error` → forwards to native via `postMessage`.
2. Native `onMessage` appends to `jsErrors` state (cap 50).
3. Red banner at bottom shows latest error in FULL (no truncation, up to 4000 chars).
4. Tap banner → full-screen `<Modal>` with `<ScrollView>` showing all errors in monospace.
5. "Copy all" button via `expo-clipboard` so user can paste errors into a chat.

Non-negotiable for shipping WebView apps to non-technical users.

## Anti-patterns

1. Don't ship all 69 features at once. Pick a wave, ship, iterate.
2. Don't add features without Settings toggles.
3. Don't duplicate PWA features in native — bridge instead.
4. Don't add native features the PWA can do.
5. Don't add `expo-background-fetch` without testing on real Android (Doze mode).
6. Don't add biometric lock without a passcode fallback.
7. Don't trust `nohup` in sandboxes — use `setsid` or foreground.
8. Don't use global `expo-cli` — use `./node_modules/.bin/expo`.
9. Don't promise an Expo Go URL in sandboxes — build the APK.
10. Don't use `expo-asset` for folders of `.js` files — Metro parses them as JS modules.

## Stop conditions (ask the user before proceeding)

1. Source has DB/filesystem access in API routes that can't move client-side.
2. Source uses Next.js features that can't be statically exported (ISR, SSR-only, image optimization).
3. Disk < 2 GB and you've cleaned everything safe.
4. No `package.json` or `next.config.{ts,js}` in source.
5. Build fails for a reason not in the bugs section, after 30 min of trying.
6. User requests a feature not in the catalog.

## Embedded scripts

### scripts/build-web-bundle.py

```python
#!/usr/bin/env python3
"""Build a TypeScript data module from a Next.js static export."""
import base64, json, re
from pathlib import Path

SOURCE_DIR = Path("/home/z/my-project/work/out")
OUTPUT_FILE = Path("/home/z/my-project/expo-app/src/web-bundle.ts")

SKIP_PATTERNS = [
    re.compile(r"^__next\."),
    re.compile(r"\.txt$"),
    re.compile(r"^404($|/)"),
    re.compile(r"^_not-found($|/)"),
    re.compile(r"^robots\.txt$"),
]

def should_skip(p): return any(x.search(p) for x in SKIP_PATTERNS)

def rewrite_paths(c):
    c = re.sub(r'((?:src|href)=["\'])/_next/', r"\1./_next/", c)
    c = re.sub(r'((?:src|href)=["\'])/(?!/)', r"\1./", c)
    c = re.sub(r'url\(["\']?/_next/', "url(./_next/", c)
    c = re.sub(r'"/_next/', '"./_next/', c)
    return c

def main():
    if not SOURCE_DIR.exists():
        raise SystemExit(f"Source not found: {SOURCE_DIR}")
    files, total, skipped = {}, 0, 0
    for root, dirs, fns in SOURCE_DIR.walk():
        dirs[:] = [d for d in dirs if not d.startswith(".") and not d.startswith("__")]
        for fn in fns:
            fp = root / fn
            rp = str(fp.relative_to(SOURCE_DIR)).replace("\\", "/")
            if should_skip(rp): skipped += 1; continue
            data = fp.read_bytes(); total += len(data)
            try:
                t = data.decode("utf-8")
                if rp.endswith((".html", ".htm")): t = rewrite_paths(t)
                files[rp] = {"type": "text", "content": t}
            except UnicodeDecodeError:
                files[rp] = {"type": "base64", "content": base64.b64encode(data).decode("ascii")}
    OUTPUT_FILE.parent.mkdir(parents=True, exist_ok=True)
    with OUTPUT_FILE.open("w", encoding="utf-8") as f:
        f.write("// AUTO-GENERATED — DO NOT EDIT.\n\n")
        f.write("export interface WebBundleFile { type: 'text' | 'base64'; content: string; }\n\n")
        f.write("export const WEB_BUNDLE: Record<string, WebBundleFile> = ")
        f.write(json.dumps(files, ensure_ascii=False, separators=(",", ":")))
        f.write(";\n")
    print(f"Wrote {OUTPUT_FILE} | {len(files)} files ({skipped} skipped) | {total:,} bytes")

if __name__ == "__main__":
    main()
```

### scripts/fix-asset-prefix-check.py

```python
#!/usr/bin/env python3
"""Patch getAssetPrefix InvariantError — indexOf('./_next/') → indexOf('_next/')."""
import json, re
from pathlib import Path

BUNDLE = Path("/home/z/my-project/expo-app/src/web-bundle.ts")

def find_obj(text):
    m = re.search(r"WEB_BUNDLE[^=]*=\s*", text)
    start = m.end(); i = start; depth = 0
    in_str = False; str_ch = None; esc = False
    while i < len(text):
        ch = text[i]
        if in_str:
            if esc: esc = False
            elif ch == "\\": esc = True
            elif ch == str_ch: in_str = False
        else:
            if ch in ('"', "'", "`"): in_str, str_ch = True, ch
            elif ch == "{": depth += 1
            elif ch == "}":
                depth -= 1
                if depth == 0: return start, i + 1
        i += 1
    raise RuntimeError("Bundle object not found")

def main():
    text = BUNDLE.read_text()
    start, end = find_obj(text)
    data = json.loads(text[start:end])
    patched = 0
    for k, v in data.items():
        if k.endswith(".js") and "Expected document.currentScript src to contain" in v.get("content", ""):
            orig = v["content"]
            v["content"] = orig.replace('t.indexOf("./_next/")', 't.indexOf("_next/")')
            if v["content"] != orig:
                patched += 1
                print(f"  patched: {k}")
    if patched == 0:
        print("No patches applied.")
        return
    new_str = json.dumps(data, ensure_ascii=False, separators=(",", ":"))
    BUNDLE.write_text(text[:start] + new_str + text[end:])
    print(f"\nPatched {patched} chunk(s). Wrote: {BUNDLE}")

if __name__ == "__main__":
    main()
```
